用纯 HTML 搭建一个博客

2026-04-17 技术

在这个框架遍地的时代，用纯 HTML 写一个博客似乎是件很复古的事。但我想说，对于个人博客这个场景，纯 HTML 可能是最优解。

为什么不用框架

我用过 Hugo、Hexo、Gatsby、Next.js 来搭建博客。它们都很好，但每次回来改点东西，都要先 npm install，然后面对一堆依赖更新的警告，有时候甚至跑不起来。

一个两三年没碰的 Hugo 项目，你敢直接改吗？一个 node_modules 比文章还大的 Next.js 博客，真的有必要吗？

博客应该能活十年以上。任何有依赖的东西，都不一定能活那么久。HTML 可以。

怎么做的

项目结构

非常简单：

my-blog/
├── index.html      ← 首页
├── about.html      ← 关于页
├── style.css       ← 样式
└── posts/
    ├── post-1.html ← 文章
    └── post-2.html

动画效果

虽然没有用任何 JS 框架，但动画效果一点也不差：

  • 页面切换：点击链接时有金色过渡动画
  • 滚动渐显：文章卡片进入视口时淡入上移
  • 鼠标光晕：一个跟随鼠标的柔和光效
  • 阅读进度：页面顶部的阅读进度条
  • 悬停效果：卡片、链接都有细腻的交互反馈

这些全都是原生 CSS 动画 + 少量 vanilla JS 实现的，没有任何第三方库。

部署

部署到 Cloudflare Pages，整个过程不到五分钟：

  1. 把文件推送到 GitHub
  2. 在 Cloudflare Pages 连接仓库
  3. Build command 留空，Build output directory 填 .
  4. Deploy

之后每次 git push，博客自动更新。

简单的东西往往最持久。这个博客没有依赖、没有构建步骤、没有 node_modules。十年后打开这个 HTML 文件，它依然能跑。
